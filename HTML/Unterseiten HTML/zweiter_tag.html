CSS Inline styling

Wird in der HTML Line des jeweiligen Elements geschrieben, als Attribute style:

            <h2 style="color: lime; background-color: black; font-family: 'Comic Sans MS'">CSS Inline styling</h2>

CSS Internal styling

Wird im Head Bereich festgelegt:

        <head>
            <style>
                h3 {
                    color: blue;
                    background-color: pink;
                    font-family: sans-serif;
                    font-size: 10pt;
                    font-style: italic;
                    }
            </style>
        </head>

CSS External styling

Wird in einer separaten CSS Datei angegeben die mittels eines <link> Element im Header verlinked wird:

        <link rel="stylesheet" href="../Styles/styles.css">

Varianten von Selektoren und deren verhalten untereinander

Hier ist ein Beispiel eines ID Selektors welcher im CSS Code mittels eines # als Präfix angesteuert wird. In diesem Fall:
                                id="id-selector"

                                #id-selector{
                                    font-family: Alef;
                                    color: darkgreen;
                                }
        

Hier ist ein Beispiel eines Class Selektors welcher im CSS Code mittels eines . als Präfix angesteuert wird. In diesem Fall:
                                class="class-selector"

                                .class-selector{
                                    background-color: red;
                                }
        

Beide Selektoren können frei kombiniert werden. Man kann sowohl einem Element mehrere Selektoren geben, als auch Selektoren spezifizieren wie pre.class-selector.
        In diesem Fall wurde dem <pre> Element sowohl der ID als auch Class Selektor gegeben. Man kann auch mittels eines Abstands eine Beziehung zwischen den auszuwählenden Elementen darstellen.
        div p würde alle p Elemente innerhalb von divs ansteuern.
        div > p würde alle p Elemente ansteuern die direkte Nachfahren von einem div Element sind.
        div > * > p würde die zweite Generation p ansteuern.
        div ~ p würde Nachbaren ansteuern.
        

Der Hierachie nach werden Selektoren anders gewichtet. Die Base ist ID > Class > Element. Die Style-Regelungen die mit einander clashen würden, werden nach Spezifizität ausgewählt.
            In diesem Fall wurde ein ID und Class Selektor gegeben:

            #id-border{
                border: 3px solid red;
            }
            .class-border{
                border: 2px solid pink;
                color: lime;
                background-color: gray;
            }

Test ob man Enkelkinder von Elementen über ihre Beziehung ansteuern kann

Das ist ein Kind.


Es können auch n-generation Elemente spezifisch ausgewählt werden mittels z.B.: td:first-child oder tr:nth-child(even)
